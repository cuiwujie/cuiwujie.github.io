javascript 正则表达式

正则表达式（英文：Regular Expression 代码中常常简写为上 regexp 或者是 RE）使用耽搁字符串来描述，匹配一系列符合某个特征句法规则的字符串搜索模式，

字符串检索

  • 在javascript中，正则表达式常用于两个字符串方法，search() 和 replace();
  • search() 方法用于检索字符串中指定的字符串，或检索与正则表达式相匹配的子字符串，并返回子字符串的起始位置。
  • replace() 方法用于在字符串中用于写字符去替换另一些字符，或换一个与正则表达式相匹配的字符的子串

  • 什么是正则表达式

  1. 正则表达式 石油一个字符序列形成的搜索模式
  2. 当你在文本中搜索数据时，你可以使用搜索模式来描述你要搜索的内容
  3. 正则表达式 可以是一个简单的字符，或者是一个更加复杂的模式
  4. 正则表达式可以用于所有文本的搜索和文本替换的工作

应用场合

数据验证 文本替换 内容替换 过滤内容

可以理解为：执行字符串函数无法完成的特殊的匹配、拆分、替换功能

创建正则表达式

在javascript正则表达式也是通过对象的方式来创建的，他有自己的方法。

  1. 通过构造函数创建 reg =new RegExp( “正则表达式” ，”模式修正符”)

    var reg= new RegExp(“wwe “);
    var stat = reg.test(“sxwwe “);
    alert(stat) ;

  2. 通过字面量方式创建

    var reg=/wwe/i;
    var stat = reg.test(“sxwwe”);
    alert(stat);

    通常将正则表达式字符串放在 /RegExp/ 中间//称为定界符

text函数

  • 返回一个Boolean值，它指出在被查找的字符串中是否存在模式

exec函数

  • RegExp.exec()在字符串中匹配正则，成功返回数组，失败返回null

  • 返回的数组包含特殊的属性：

  • input//被查找的字符串 index //字符串的位置
  • 如果采用g修饰
  • 如果正则表达式没有设置g修饰，那么 exec方法不会对正则表达式产生有任何的影响，如果设置了g，那么执行exec之后会更新正则表达式的lastIndex属性，表示本次匹配后
    ，所匹配的字符串的下一个字符的索引，下一次使用这个正则表达式匹配字符串的时候就会从上一次的lastIndex属性开始匹配，

正则表达式知识点

原子

什么是原子

\d  原子时正则表达式中的最小的元素，包括英文，标点符号等
\d  匹配任意一个数字[0-9]
\D  与除了数字以外的任何一个字符匹配[^0-9]
\w    与任意一个英文字母，数字或下划线匹配[a-z0-9A-Z_]
\W  除了字母，数字或下划线外语任何一个字符匹配[^a-z0-9A-Z_]
\s  与任意一个空白字符匹配

\f 换页字符
\n 换行字符
\r 回车字符
\t 制表符
\v 垂直制表符
\S 与除了空白符符外任意一个字符匹配[^\n\f\r\t\v]

原子表

[]            只匹配其中的一个原子
[^]            只匹配除了其中字符的任意一个原子
[0-9]        只匹配0-9任何一个数字
[a-z]        只匹配小写字母a-z任何一个字母
[A-Z]        只匹配da写字母a-z任何一个字母

元字符

在正则表达式中有一些特殊的字符代表特殊意义叫元字符

  • . 除换行符意外的任何一个字符
  • | 或的意思，匹配其中一项就代表匹配
  • 匹配新版身份证号
  • /^\d{18}$/

原子分组

匹配多个字符时用()分组，分组代表一个原子集合或者是说一个大原子，并压入堆栈（内存）用于调用，组号是要从左到右计数的调用时候：如果字面量形式用\1,构造函数方式用\1这种方式我们叫做反向引用
例：

var reg = new RegExp("(hdw)123\\1","i");
alert(reg.test("hdw123hdw"));

量词

可以使用一些元字符，重复一些元子或元字符

    *         重复零次或多次
    +         重复一次或多次
    ?         重复零次或一次
    {n}       重复n次
    {n,}      重复n次或多次
    {n,m}     重复n次到m次

举例

    一片两片三四片 ，落尽正则全找见

    {1}，{2}，{3,4}，{1，}。

贪婪和吝啬

正则匹配是贪婪的，禁止贪婪用 ？

*？        重复零次，但尽可能少重复
+？        重复一次或多次，但尽可能少重复
?？        重复零次或一次，但尽可能少重复
{n,}？     重复n次或多次，但尽可能少重复
{n,m}？    重复n次到m次，但尽可能少重复

匹配边界

^         匹配字符串的开始
$         匹配字符串的结束，忽略换行符
\b         匹配单词的边界
\B         匹配出单词边界以外的部分

模式修正符

i        不区分大小写的字母的匹配
m        将字符串视为多行，修饰^ and $
g        全局匹配，找到所有匹配项

模式匹配的顺序

模式匹配的顺序（从高到低）

顺序            元字符                说明

1            （ ）                模式单元
2            ？ * + {}            重复匹配
3            ^ $                    边界限制
4            |                    模式选择

字符串中用到的正则表达式

str.search(regexp)
str.replace();

  • regexp为正则表达式，泛会所因为之，不支持全局索引找到即停止搜索

    例子：

    var str = "www.zktl123.com";

    alert(str.search(/zktl/));
