webpack

关于前端自动化，也是看了几篇帖子感觉比较神奇，因为平时实验室做项目,并没有所谓的前后端的分离，也不怎么用的到。以前倒是接触了一些Grunt，了解了一下它的基本的用法，把一些插件安装上，简单的体会了一下，后来很长时间就再也咩有接触它。这不学到了框架Vue，好多都是与自动化相结合的，所以索性就好好的深入研究一下吧。我在这里只是针对这个过程步骤说明了一下。

神马是webpack

先说说这个问题，老生常谈了，这也是属于，他就是一款模块加载器兼打包工具，他可以把各种资源，例如js（JSX），coffee,样式（含有less和sass，图片等作为模块来使用和处理。

我们可以使用require（XXX）的形式来引入各个模块，及时它们可能需要经过编译（比如Jsx和sass），但是我们无需在上面花费太多的心思，因为webpack有着各种健全的加载器（loader）在默默的处理这些事情。

webpack的官网http://webpack.github.io，文档地址是http://webpack.github.io/docs,深入的文档如果你想更为详细的了解一下可以点击进去查一查。

webpack有什么优势

  1. 它是以commonJs的形式来书写的，但是ＡＭＤ／ＣＭＤ的支持也可以，也方便旧的项目的移植。
  2. 不能模块化的不仅仅是JS了
  3. 开发便捷方便，能够替代部分grunt/gulp的工作，比如打包，压缩混淆、图片转base64等。
  4. 扩展性能强，插件机制完善，特别支持react的热插拔
  5. 废话不多说了直接开始学习吧

下一步我们该在webpack.config.js文件中来配置coffee-loader的一些参数这样我们才能正常的使用它。

module.exports = {
entry: './main.js',
output: {
    filename: 'bundle.js' 
},
module: {
    loaders: [
    { test: /\.coffee$/, loader: "coffee-loader" }
    ]
},
resolve: {
    extensions: ['', '.js', '.jsx']
},
plugins: []
};

解释config文件中的内容

  1. entry 是页面中的入口文件，比如我这边的入口文件时main.js

  2. output: 是指页面通过webpack打包后生成的目标文件放在什么地方去，我这边是在根目录下生成build文件夹，该文件夹内有一个build.js文件；

  3. resolve: 定义了解析模块路径时的配置，常用的就是extensions; 可以用来指定模块的后缀，这样在引入模块时就不需要写后缀，会自动补全。

  4. plugins: 定义了需要使用的插件，比如commonsPlugin在打包多个入口文件时会提取公用的部分，生成common.js;

  5. module.loaders：是文件的加载器，比如我们之前react需要在页面中引入jsx的js源码到页面上来，然后使用该语法，但是通过webpack打包后就不需要再引入JSXTransformer.js；看到上面的加载器；比如jsx-loader加载器就是代表JSXTransformer.js的，还有style-loader和css-loader加载器；因此在使用之前我们需要通过命令把它引入到项目上来；因此需要如下命令生成下；

理解webpack加载器

Webpack提供了一套加载器，比如css-loader,less-loader,style-loader，url-loader等，用于将不同的文件加载到js文件中，比如url-loader用于在js中加载png/jpg格式的图片文件，css/style loader用于加载css文件，less-loader加载器是将less编译成css文件；

了解下webpack的几个命令

  1. webpack // 最基本的启动webpack的方法
  2. webpack -w // 提供watch方法；实时进行打包更新
  3. webpack -p // 对打包后的文件进行压缩
  4. webpack -d // 提供source map，方便调式代码

如何独立打包成样式文件

有时候我们不想把样式打在脚本中，而是想独立css出来，然后在页面上外链css，这时候我们需要 extract-text-webpack-plugin 来帮忙：

我们首先需要安装 extract-text-webpack-plugin：如下： 

npm install extract-text-webpack-plugin –save-dev

现在我们需要看看webpack.config.js 配置变成如下：

var ExtractTextPlugin = require("extract-text-webpack-plugin");
// 使用webpack打包
module.exports = {
  entry: "./src/main.js",
  output: {
    filename: "build.js"
  },
  module: {
    loaders: [
      //.css 文件使用 style-loader 和 css-loader 来处理
      {
        test: /\.less$/,
        loader: ExtractTextPlugin.extract(
            'css?sourceMap!' +
            'less?sourceMap'
        )
      }
    ]
  },
  resolve: {
    extensions: ['', '.js', '.jsx']
  },
  // 内联css提取到单独的styles的css
  plugins: [new ExtractTextPlugin('styles.css')]
};

配置完成后 我们webpack运行下即可，在build文件夹内会生成2个文件，一个是build.js 处理模块的文件 另一个就是我们的styles.css了；这个时候我们就可以分别引用了。
